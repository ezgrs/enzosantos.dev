Tudo começou com alguns erros de compilação que não faziam muito sentido para mim.

Investigando o problema, descobri que eu entendia bem menos do que imaginava sobre target, module e moduleResolution no tsconfig.json, o arquivo que define como o TypeScript analisa e gera o JavaScript da aplicação.

A pesquisa acabou me levando por temas como ESM, CommonJS, Node.js, bundlers e resolução de módulos. Neste post, compartilho um resumo do que aprendi e das combinações mais comuns encontradas em projetos modernos.

## module

Controla o formato dos _imports_ e _exports_ gerados.

Os principais valores podem ser:

- **commonjs** para projetos Node antigos
- **es2020** para ESM padrão
- **es2022** para ESM moderno
- **esnext** para sintaxe mais recente
- **node20** para Node moderno
- **nodenext** para Node mais recente

Considere o seguinte código em Typescript:

```typescript
import express from "express";
```

Ao definir o *tsconfig.json* para

```json
{
    "module": "commonjs",
}
```

o código JavaScript gerado será

```javascript
const express = require("express");
```

Ao definir o *tsconfig.json* para

```json
{
    "module": "esnext",
}
```

o código JavaScript gerado será

```javascript
import express from "express";
```
